# ST News Publisher

このスキルは、ST Channelにおけるニュース公開プロセスを標準化し、**人間が書いたような深く、熱く、ためになるテック記事**を自動生成するための手順書です。

## 概要

このスキルを使用して、以下のタスクを実行します：
1.  **週次管理**: 現在の日付に基づき、適切な週次フォルダを特定または作成する。
2.  **ニュース選定**: Inbox内のRSSリストをスコアリングし、上位記事を抽出する。
3.  **Deep Dive記事作成**: 選定記事を「脱AI・人間味あふれる10のポイント」に基づき、**大学生〜専門家レベルの深さ（2000文字以上）**で執筆する。
4.  **タイトル最適化**: 3案を作成し、最も惹きつけられる1案を採用する。
5.  **まとめ更新**: 週次まとめファイル（Folder Note）に記事リンクを追加する。
6.  **デプロイ**: 変更をGitHubにプッシュし、公開サイトへ反映する。

## 使い方 (Usage)

> 「ニュースを更新して（Deep Diveで）」
> 「InboxのRSSから長文記事を作成して」

このスキルを呼び出すことで、ターゲットフォルダの計算からGit Pushまでの一連の流れを自律的に実行できる。

## 前提条件

- カレントディレクトリが `10_Projects/ST_channnel`（リポジトリルート）であること。
- `01_News/Inbox` にRSS収集済みの未処理ファイルが存在すること。

## 手順 (Workflow)

（基本フローは変更なし。ただし、記事作成フェーズでの品質基準が厳格化されている）

### 1. ターゲットフォルダの特定

```bash
python3 .agent/skills/ST_News_Publisher/scripts/get_weekly_target.py
```

### 2. ニュース選定と記事執筆

1.  **パイプライン実行**

```bash
python3 .agent/skills/ST_News_Publisher/scripts/run_news_pipeline.py
```

2.  **記事執筆（Deep Dive Mode）**
    - 各ニュースについて、**必ずリンク先の本文をツールで取得・熟読し**、`TARGET_DIR` 内に日刊記事ファイル (`YYYY-MM-DD_Title.md`) を作成する。
    - 内容は `.agent/skills/ST_News_Publisher/assets/templates/Daily_News_Template.md` の構成に従うこと。
    - **脱AI・人間味あふれる10のポイント**を遵守しつつ、**文字数は2000文字以上**を目指す。
    - 技術的なメカニズム（Why/How）を徹底的に掘り下げ、表面的な要約で終わらせないこと。
    - 専門用語は初出で補足を付けるが、子供だましな説明は避け、知的好奇心を満たすレベルを維持する。

3.  **タイトル最適化**

```bash
python3 .agent/skills/ST_News_Publisher/scripts/title_optimizer.py \
  --fact "<事実>" \
  --surprise "<驚き>" \
  --specific "<具体性>"
```

### 3. まとめファイルの更新

作成した記事を `TARGET_DIR/YYYY-MM-DD--ST-news.md` の一覧テーブルに追加する。

### 4. 変更の公開 (Deploy)

```bash
git add .
git commit -m "Update news: YYYY-MM-DD"
git push origin main
```

## 脱AI・人間味あふれる10のポイント（Deep Dive Edition）

1.  **教科書感の排除**: 語りかけるように書く。
2.  **リズムの変化**: 独白、倒置、体言止めを駆使する。
3.  **感情の注入**: 書き手の感動や衝撃を隠さない。
4.  **会話的な調子**: 読者との対話を意識する。
5.  **共感の獲得**: 読者の疑問を代弁する。
6.  **具体化**: 比喩は「日常的なもの」と「技術的な正確さ」を両立させる。
7.  **人間味の演出**: 完璧すぎない、思考のプロセスを見せる。
8.  **中立病の打破**: ポジションを取る。賛成か反対か、期待か懸念か。
9.  **繰り返しの回避**: 語尾のバリエーションを持たせる。
10. **最後の問い**: 読者の思考を未来へ飛ばす。
