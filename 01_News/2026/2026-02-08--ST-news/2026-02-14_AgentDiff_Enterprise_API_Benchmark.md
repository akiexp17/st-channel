# Enterprise APIタスクでエージェントを比較、Agent-Diffが状態差分評価を導入

> [!tip] タイトル候補（事実＋驚き＋具体性）
> 1. Enterprise APIタスクでエージェントを比較、Agent-Diffが状態差分評価を導入
> 2. API実行結果を状態変化で採点、Agent-Diffがコード実行型評価を整理
> 3. エージェントの「手順ミス」を可視化、Agent-DiffがState-Diff評価を提示
> 採用理由: 企業API実務との接続点と評価手法（State-Diff）が明確なため。

# 30秒サマリー
Agent-Diffは、Enterprise API操作をコード実行で再現し、最終出力だけでなく状態差分（State-Diff）でエージェントを評価する枠組みを示した。中間操作の誤りを追跡しやすく、業務自動化の品質検証を具体化する。

# 何が起きた
企業APIタスクに特化したエージェント評価研究が公開された。実行過程をコードで検証し、前後状態の差分を使って正否を判定する方式を採用している。

# なぜ重要か
API連携の失敗は「一見成功だが状態が壊れる」形で起きやすい。状態差分評価を導入すると、最終レスポンスだけでは見えない不整合を早期に検出できる。

# 技術ポイント
- 企業向けAPIタスクを対象にした評価である（根拠: タイトルに`Enterprise API Tasks`）
- コード実行を前提にした検証方式を採用している（根拠: タイトルに`via Code Execution`）
- 評価軸として状態差分を明示している（根拠: タイトルに`State-Diff-Based Evaluation`）

# 懐疑点・未確定要素
- 実企業の認証・権限制約をどこまで再現できるかは不明。
- API仕様変更が頻繁な環境でのベンチ維持コストは未検証。

# 実務インパクト
- API自動化エージェントの受け入れ試験を、状態整合チェック中心に設計できる。
- 障害時の原因切り分け（入力/手順/状態反映）を標準化しやすい。
